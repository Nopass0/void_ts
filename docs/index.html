Official TypeScript client for VoidDB with typed collections, schema pull and push, generated model definitions, relation includes, cache helpers, and a query builder that stays close to the underlying HTTP API.

View repository Open core server Server docs

Typed CRUD

Use collection-level generics for inserts, reads, updates, and counts without wrapping your app in a large framework.

Schema workflow

Pull live schema from VoidDB, diff it, push changes back, and pair it with the core migration flow.

Type generation

Generate `.d.ts` model files from a running VoidDB instance and keep application types aligned with production data.

Relation includes

Fetch linked documents with explicit include clauses for dashboards, APIs, and admin tools.

Install

Works with Node.js and Bun. Pair it with a running VoidDB server.

npm install @voiddb/orm
bun add @voiddb/orm

Connect and query

Login and CRUD

import { VoidClient, query } from "@voiddb/orm";

type User = {
  _id: string;
  name: string;
  age: number;
  active: boolean;
};

const client = new VoidClient({ url: "http://localhost:7700" });
await client.login("admin", "admin");

const users = client.db("app").collection<User>("users");

const id = await users.insert({
  name: "Alice",
  age: 30,
  active: true,
});

const user = await users.get(id);
await users.patch(id, { age: 31 });
await users.delete(id);

Query builder

const adults = await users.find(
  query()
    .where("age", "gte", 18)
    .where("active", "eq", true)
    .orderBy("created_at", "desc")
    .limit(50)
);

Relation includes

Use explicit include clauses when your schema exposes relation metadata.

const result = await client
  .db("app")
  .collection("users")
  .findWithRelations<{
    profile: { _id: string; bio: string };
  }>(
    query().where("_id", "eq", "user-1"),
    [
      {
        as: "profile",
        relation: "many_to_one",
        target_col: "profiles",
        local_key: "profile_id",
        foreign_key: "_id",
      },
    ]
  );

Schema pull, push, and planning

const project = await client.schema.pull();
const plan = await client.schema.plan(project, {
  dryRun: true,
});

for (const op of plan.operations) {
  console.log(op.summary);
}

await client.schema.push(project, {
  forceDrop: false,
});

This pairs naturally with the core server CLI:

voidcli schema pull --out void.prisma
voidcli schema push --schema void.prisma
voidcli migrate dev --schema void.prisma --name init
voidcli migrate deploy --dir void/migrations

Generate model types

const project = await client.schema.pull();
const dts = client.schema.generateTypes(project, {
  moduleName: "@acme/void-models",
});

console.log(dts);
npx voiddb-orm generate \
  --url http://localhost:7700 \
  --username admin \
  --password admin \
  --output ./src/generated/voiddb-models.d.ts

  --username admin \
  --password admin

bunx voiddb-orm migrate dev \
  --schema ./void.schema.prisma \
  --name add_users

npx voiddb-orm migrate status

What else is included

  • Access token and refresh token handling through the built-in HTTP client.
  • Cache helpers for key-value workflows such as sessions and temporary computed state.
  • A small public API surface that maps closely to VoidDB server endpoints.
  • Easy pairing with the main admin panel, schema files, and PostgreSQL import workflow.

VoidDB ORM is built for the core server at github.com/Nopass0/void.
